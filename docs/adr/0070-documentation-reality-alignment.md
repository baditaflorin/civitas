# 0070 - Documentation-Reality Alignment Process

## Status

Accepted

## Context

README claims overstate OCR/transcription/indexing compared with current processor-needed behavior.

## Decision

Each user-facing README claim must map to one of:

- An implemented and tested behavior.
- A documented limitation.
- A removed claim.

Update README, API docs, privacy docs, and postmortem when behavior changes.

## Consequences

Marketing language becomes less grand but more trustworthy.

## Alternatives Considered

Keeping aspirational README claims was rejected because Phase 3 specifically targets stranger usability.
